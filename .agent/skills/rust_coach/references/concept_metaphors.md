# Rust 개념 비유 모음

## 메모리 & 소유권
- **소유권(Ownership)**: 집문서. 한 명만 가질 수 있고, 넘기면 원래 주인은 더 이상 쓸 수 없음.
- **이동(Move)**: 집문서 명의 이전. 주소(메모리)는 그대로, 소유자만 바뀜.
- **참조/대여(Borrowing, &)**: 집 열쇠를 잠깐 빌려주기. 원본 소유권은 유지.
- **가변 참조(&mut)**: 집 수리를 허락하는 독점 열쇠. 동시에 두 명에게 줄 수 없음.

## 타입 시스템
- **&str**: 비석에 새긴 각인. 읽기 전용, 불변, 빠름.
- **String**: 이동식 칠판. 힙에 할당되며 수정 가능.
- **Copy 트레이트**: 책 vs 집문서. 숫자처럼 복사가 쉬운 것은 소유권 이동 없이 복사됨.

## 타입 추상화
- **트레이트(Trait)**: 특정 능력을 가진 자격증. 자격증만 있으면 누구든 그 능력을 쓸 수 있음.
- **제네릭(Generic)**: 어떤 모양의 물건이든 들어갈 수 있는 만능 상자.
- **Option<T>**: 선물 상자. 열어보기 전까지 안에 뭔가 있을 수도(Some), 없을 수도(None) 있음.
- **Result<T, E>**: 택배 상자. 성공(Ok)이면 물건이 들어있고, 실패(Err)면 오류 메모가 들어있음.

## 자료구조
- **Vec**: 번호표가 붙은 줄. 끝에 추가하거나 꺼내기가 빠름.
- **HashMap**: 이름표가 붙은 보관함. 키로 바로 해당 칸을 찾아가는 방식.
- **Entry API (.entry().or_default())**: "있으면 가져오고, 없으면 새로 만들어라"를 한 줄로.

## 코드 흐름
- **Iterator (반복자)**: 게으른 일꾼. 시킬 때까지 실제로 일하지 않음(지연 평가).
- **클로저(Closure, |x| ...)**: 그 자리에서 즉석으로 만드는 1회용 함수.
- **match 문**: 신분 확인 게이트. 확인 없이는 데이터에 손댈 수 없음.

## 구조
- **구조체(Struct)**: AND의 세계. A와 B와 C가 모두 함께 존재.
- **열거형(Enum)**: OR의 세계. A 또는 B 또는 C 중 딱 하나만 존재.
- **모듈(Module)**: 건물 내부 구조. 기본은 닫힌 문(private), pub으로 열어줘야 밖에서 접근 가능.
- **매크로(Macro, !)**: 컴파일 타임 마법사. 실행 전에 짧은 코드를 긴 코드로 미리 펼쳐줌.
